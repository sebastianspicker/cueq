### Catalog (+9 new tests, + testability refactor)

- `effectiveTo` inclusive boundary: asOf equals effectiveTo → included
- `effectiveTo` exclusive: asOf after effectiveTo → excluded
- `effectiveTo` within range: asOf between effectiveFrom and effectiveTo → included
- **Multi-version resolution**: latest version wins (v3 over v1, v2)
- **Per-type version independence**: BREAK v2 and REST v5 resolved separately
- **Expired + current**: expired v1 excluded, current v2 included
- `getPolicyHistory` version-descending ordering with 3 versions
- `getPolicyHistory` type filtering across mixed types
- `getPolicyHistory` without filter returns alphabetical sort

### Evaluation Types (+5 new tests)

- `ruleVersion=0` accepted (schema uses `.int()` not `.int().positive()`)
- Non-integer `ruleVersion` rejected
- Multiple violations in single `PolicyEvalResult`
- Missing `ruleId` rejected
- Missing `message` rejected
