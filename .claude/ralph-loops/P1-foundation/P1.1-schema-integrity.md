# P1.1: Schema Integrity Audit

You are auditing the schema layer of this cueq repository. Work through items ONE AT A TIME.

The schema layer has two tiers:

1. JSON Schemas in `schemas/domain/` and `schemas/fixtures/` (30+ files)
2. Zod schemas in `packages/shared/src/schemas/` (12 files)

These MUST be consistent with each other and with the Prisma model in
`packages/database/prisma/schema.prisma` (25+ models, 10+ enums).

## What To Do

For each JSON Schema file in `schemas/domain/`:

1. Read the schema and identify every field, type, enum, and constraint
2. Read the corresponding Zod schema in `packages/shared/src/schemas/`
3. Read the corresponding Prisma model in `packages/database/prisma/schema.prisma`
4. Flag any mismatches: missing fields, type differences, enum value differences,
   constraint mismatches (e.g. nullable vs required)
5. Fix mismatches by aligning to the Prisma model as source of truth for data shape,
   and the JSON Schema as source of truth for validation constraints

After fixing, run `make schemas` to validate JSON Schemas compile.

## What NOT To Do

- Do NOT add new schemas, models, or fields that don't already exist
- Do NOT change business logic in `packages/core/` or `packages/policy/`
- Do NOT touch controllers, services, or frontend code
- Do NOT modify test files (only schema/type files)

## Verification

After each fix: `make schemas && make typecheck`
