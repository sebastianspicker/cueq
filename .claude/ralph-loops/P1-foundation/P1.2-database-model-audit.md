You are auditing the Prisma schema for the cueq database. Work through items ONE AT A TIME.

The schema is at `packages/database/prisma/schema.prisma` (25+ models, 10+ enums).

## What To Do

1. Check every model for proper indexing:
   - Foreign keys should have indexes (Prisma adds implicit ones for @relation, but
     check composite queries)
   - Frequently queried date ranges (personId + startTime patterns) should have
     composite indexes
   - The AuditEntry table must NOT have updatedAt (it's append-only)

2. Check enum consistency:
   - Every enum value should be consistent with values used in
     `packages/core/src/core/` and `packages/shared/src/schemas/`
   - Enum values should match the AbsenceType, WorkflowType, BookingSource etc.

3. Check referential integrity patterns:
   - Cascade delete rules: are they appropriate? (AuditEntry should NEVER cascade delete)
   - Optional vs required relations: do they match domain rules?
   - Any deprecated fields still referenced anywhere?

4. Check for missing constraints:
   - @@unique constraints that protect domain invariants
   - Check boundaries (Decimal precision for hours/money)

5. Review database test files in `packages/database/src/__tests__/`:
   - Does `database.smoke.test.ts` test basic connectivity?
   - Does `database.compliance.test.ts` verify audit immutability at DB level?
   - Does `database.integration.test.ts` cover key queries?
