# P4.2: Controller Security & Input Validation Audit

You are auditing every API controller for security and input validation.
Work through ONE controller at a time.

Controllers are in `apps/api/src/phase2/controllers/` (18 files).

Auth guards: `apps/api/src/common/guards/`
Auth service: `apps/api/src/common/auth/`
DTOs: `apps/api/src/phase2/dto/`

## What To Do

For each of the 18 controllers:

1. Verify every endpoint has appropriate `@Roles()` decorator matching the
   product spec access control matrix
2. Verify every input parameter is validated through Zod schemas from
   `@cueq/shared` (no raw `any` or unvalidated body parsing)
3. Verify ID parameters are validated as CUID format
4. Verify date parameters are validated as ISO-8601
5. Verify pagination parameters have upper bounds (max 100)
6. Check that error responses follow consistent format with correlation IDs
7. Verify no endpoint leaks data across role boundaries:
   - Employees should not see other employees' absence reasons
   - Team leads should only see their own team
   - Payroll role sees aggregate data, not individual patterns

## What NOT To Do

- Do NOT modify service logic
- Do NOT change database models or schemas
- Do NOT modify frontend code
- Do NOT add new endpoints
