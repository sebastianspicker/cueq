## What To Do

For each oversized service (>400 lines):

1. Read the entire file and identify distinct responsibility clusters
2. Extract cohesive clusters into focused helper modules under
   `apps/api/src/phase2/helpers/` or dedicated service files
3. The original service should become a thin orchestrator that delegates
4. Ensure the public API (method signatures) doesn't change
5. Keep extracted modules under 400 lines each

Patterns to look for:

- Groups of private methods that always work together
- Data transformation/mapping logic that can be pure functions
- Validation logic that can be extracted
- Repeated Prisma query patterns that can be repository helpers

## What NOT To Do

- Do NOT change the controller layer (method signatures must remain stable)
- Do NOT modify core domain logic in `packages/core/`
- Do NOT change schemas or types
- Do NOT modify test files unless imports change
- Each commit should be <400 lines changed
