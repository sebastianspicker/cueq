## Audit Findings

### Already Solid (no changes needed)

- All third-party actions SHA-pinned with version comments
- `permissions` properly scoped in all 3 workflows
- All shell scripts use `set -euo pipefail`
- File paths properly quoted throughout
- Node/pnpm versions consistent (Node 20, pnpm 9.15.0)
- Node scripts have proper error handling (catch + process.exit(1))
- `make setup` handles Docker failures gracefully (SKIP_DOCKER, retry logic)
- pnpm store cached via `cache: pnpm` in setup-node

### Issues Fixed

1. **codeql.yml missing concurrency** — Added concurrency group with `cancel-in-progress` only for PRs (scheduled runs complete fully).

2. **backup-restore-weekly.yml missing concurrency** — Added concurrency group with `cancel-in-progress: false` (never cancel weekly drills).

3. **pnpm audit `|| true` silently succeeds** — Changed to `continue-on-error: true` at step level so failures show as yellow warnings in GitHub Actions UI instead of green success.

4. **No turbo cache in CI** — Added `actions/cache@v4.2.3` for `.turbo` directory in the validate job, keyed on OS + lockfile + SHA with restore-keys fallback. Not added to fresh-clone-smoke (intentionally tests cold builds).

5. **generate-core-schema-types.mjs non-deterministic** — Removed `Generated at: <timestamp>` from output. `make generate` now produces identical output when schemas haven't changed.

6. **generate-db-schema-doc.mjs non-deterministic** — Removed `Last generated: <timestamp>` from output. Same determinism benefit.

7. **export-openapi.mjs missing error handling** — Wrapped top-level await in `main().catch()` pattern with `try/finally` to ensure `app.close()` always runs. Matches the error handling pattern used by all other Node scripts.

8. **Makefile `clean` uses hardcoded `docker-compose`** — Added inline detection: uses `docker-compose` if available, falls back to `docker compose` (V2 plugin). Matches the `docker_compose_cmd` pattern in `lib.sh`.

9. **Duplicate `docs:links` check in CI** — Removed standalone "Docs Link Check" step from ci.yml since `make check` (via check.sh) already runs `docs:links`.

## Verification

`make check` passes cleanly after all fixes.
