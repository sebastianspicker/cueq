## Bugs Found

None. The minute-by-minute cursor approach in `evaluateTimeRules` correctly handles all surcharge category transitions at midnight boundaries. Rest period calculations use pure UTC arithmetic, making them DST-safe by design. Holiday matching correctly uses local (Berlin) date from `localMinuteInfo`, not the UTC date. Break threshold comparisons use strict `<` (not `<=`), meaning "exactly sufficient" break is compliant — correct per ArbZG.
