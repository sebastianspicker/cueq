# P2.1: Time Engine Edge Cases

You are hardening the time engine in `packages/core/src/core/time-engine/`.
Work through items ONE AT A TIME.

This is pure domain logic with zero I/O dependencies. It handles:

- Flextime balance calculation (`flextime.ts`)
- Surcharge calculation for nights/weekends/holidays (`surcharge.ts`)
- Plausibility checks for booking intervals (`plausibility.ts`)
- On-call rest period validation (`oncall-rest.ts`)

Read the product spec at `docs/product-specs/time-engine-rules.md` for requirements.
Read existing tests at `packages/core/src/core/time-engine/__tests__/`.
Read reference fixtures at `fixtures/reference-calculations/` for expected behaviors.

## What To Do

For each module (flextime, surcharge, plausibility, oncall-rest):

1. Read the code and understand every code path
2. Read existing tests and identify uncovered edge cases
3. Add tests for:
   - Boundary conditions (exactly 6h = break required, exactly 9h = max daily, midnight crossings)
   - Empty/minimal inputs (no bookings, single booking)
   - Overlapping intervals
   - DST transitions (clock change dates in NRW)
   - Holiday + weekend surcharge stacking
   - Part-time workers (weeklyHours < 39.83)
   - Negative/zero duration intervals
4. Fix any bugs discovered through edge case testing
